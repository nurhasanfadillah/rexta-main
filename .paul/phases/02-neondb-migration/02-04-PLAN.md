## Goal
Buat dua Vercel Functions: `api/transactions.ts` (GET paginated+filter + POST via stored procedure) dan `api/dashboard.ts` (GET aggregate stats), keduanya protected via `requireSession`.

## Purpose
Transactions adalah operasi paling kritis di app — atomic update stok + insert record via stored procedure `process_inventory_transaction`. Dashboard menyediakan ringkasan aggregate (total items, nilai aset, transaksi terbaru) yang dibutuhkan `DashboardView.tsx`. Tanpa keduanya, Plan 02-05 (Frontend migration) tidak bisa dipetakan ke API yang lengkap.

## Output

  <name>Task 1: Buat Transactions API</name>
  <files>api/transactions.ts</files>
  <action>
    Buat file `api/transactions.ts`:

    1. Import: `import { db } from '../lib/db'` dan `import { requireSession } from '../lib/auth-middleware'`

    2. Buat `mapTransactionFromDB(row)`:
       ```
       {
         id: row.id,
         itemId: row.item_id,
         itemType: row.item_type,
         type: row.type,
         qty: Number(row.qty),
         date: row.date,
         notes: row.notes || null,
         balanceAfter: Number(row.balance_after)
       }
       ```

    3. `export default async function handler(req, res)`:
       - Jalankan `await requireSession(req)` di awal, catch → return 401

    4. GET handler:
       - Parse: `page` (default 1), `limit` (default 20), `itemId`, `itemType`, `dateFrom`, `dateTo`
       - Dynamic WHERE: conditions[], params[], idx++
         - `itemId` ada → `item_id = $N`
         - `itemType` ada → `item_type = $N`
         - `dateFrom` ada → `date >= $N`
         - `dateTo` ada → `date <= $N`
       - Parallel: COUNT + SELECT * ORDER BY date DESC LIMIT/OFFSET
       - Return `{ data: rows.map(mapTransactionFromDB), count }`

    5. POST handler — panggil stored procedure:
       - Destructure body: `{ id, item_id, item_type, type, qty, date, notes, manual_stock }`
       - Panggil stored procedure:
         ```
         SELECT process_inventory_transaction($1,$2,$3,$4,$5,$6,$7,$8) as new_stock
         params: [id, item_id, item_type, type, qty, date, notes || null, manual_stock ?? null]
         ```
       - Stored procedure returns new_stock (numeric)
       - Fetch transaction yang baru dibuat: SELECT * FROM transactions WHERE id = $1
       - Return `{ transaction: mapTransactionFromDB(txRow), newStock: Number(new_stock) }`

    6. Fallback: return 405

    Error handling: try/catch per method, prefix `[transactions]`.

    Avoid:
    - Jangan coba INSERT langsung ke transactions — selalu gunakan stored procedure untuk menjaga atomicity stok update
    - `manual_stock` hanya dipakai untuk type='OPNAME' — kirim null untuk tipe lain (stored procedure menghandle ini)
    - Jangan skip fetch transaction setelah stored procedure — frontend butuh object lengkap

<task type="auto">
  <name>Task 2: Buat Dashboard API</name>
  <files>api/dashboard.ts</files>
  <action>
    Buat file `api/dashboard.ts`:

    1. Import: `import { db } from '../lib/db'` dan `import { requireSession } from '../lib/auth-middleware'`

    2. `export default async function handler(req, res)`:
       - Jalankan `await requireSession(req)` di awal, catch → return 401

    3. Hanya GET handler:
       - Jalankan 2 query paralel via Promise.all:

         Query 1 — aggregate stats (satu query):
         ```sql
         SELECT
           (SELECT COUNT(*) FROM products)::int as total_products,
           (SELECT COUNT(*) FROM materials)::int as total_materials,
           (SELECT COALESCE(SUM(stock * price_cmt), 0) FROM products) as product_asset_value,
           (SELECT COALESCE(SUM(stock * price), 0) FROM materials) as material_asset_value
         ```

         Query 2 — recent transactions (10 terbaru):
         ```sql
         SELECT * FROM transactions ORDER BY date DESC LIMIT 10
         ```

       - Hitung `totalAssetValue = productAssetValue + materialAssetValue`

       - Return:
         ```json
         {
           "totalProducts": number,
           "totalMaterials": number,
           "productAssetValue": number,
           "materialAssetValue": number,
           "totalAssetValue": number,
           "recentTransactions": Transaction[]
         }
         ```
         Semua numeric field di-parse dengan Number() untuk menghindari string dari PostgreSQL.

    4. Fallback: return 405

    Error handling: satu try/catch untuk GET, prefix `[dashboard]`.

    Avoid:
    - Jangan gunakan N+1 queries — semua aggregate dalam satu query
    - Jangan return raw DB rows — map recentTransactions dengan mapTransactionFromDB
    - mapTransactionFromDB harus didefinisikan lokal di file ini (tidak import dari transactions.ts)
  </action>
