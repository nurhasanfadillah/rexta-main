<objective>
## Goal
Bangun foundation Drizzle ORM: install packages, buat `lib/schema.ts` sebagai single source of truth untuk semua tabel, dan rewrite `lib/db.ts` untuk export Drizzle instance (bukan raw Pool).

## Purpose
Semua plan berikutnya (auth, CRUD APIs, transactions) bergantung pada dua export dari foundation ini: `db` (Drizzle instance) dan tipe-tipe schema. Tanpa plan ini, tidak ada yang bisa direwrite. Foundation yang benar di sini juga fix masalah utama: multiple Pool instances tersebar di `lib/auth.ts` dan `api/auth-proxy.ts` — setelah plan ini, satu Pool dipakai via Drizzle.

## Output
- `package.json` — tambah `drizzle-orm` dependency, `drizzle-kit` devDependency, script `db:push`
- `drizzle.config.ts` — konfigurasi drizzle-kit
- `lib/schema.ts` — BARU: Drizzle schema untuk semua 8 tabel (4 domain + 4 Better Auth)
- `lib/db.ts` — REWRITE: export `db` (Drizzle instance) menggantikan raw Pool + neon sql
</objective>

<task type="auto">
  <name>Task 2: Buat lib/schema.ts — full Drizzle schema semua 8 tabel</name>
  <files>lib/schema.ts</files>
  <action>
    Buat file `lib/schema.ts` dengan schema Drizzle yang **exact match** dengan `db_schema_neon.sql`.
    Ini adalah source of truth baru — gantikan `db_schema_neon.sql` untuk development going forward.

    **Penting — perhatian khusus untuk Better Auth tables:**
    - Better Auth menggunakan camelCase column names di DB (bukan snake_case):
      `emailVerified`, `createdAt`, `updatedAt`, `userId`, `accountId`, `providerId`, dll.
    - Domain tables menggunakan snake_case: `category_id`, `price_cmt`, `is_favorite`, `balance_after`, dll.
    - Drizzle column definition: `columnName: type('db_column_name')` — pertama adalah TS field name, dalam string adalah nama kolom di DB.

    **Isi `lib/schema.ts`:**

    ```typescript
    import {
      pgTable,
      text,
      numeric,
      boolean,
      timestamp,
    } from 'drizzle-orm/pg-core';

    // ─── Domain Tables ────────────────────────────────────────────────────────

    export const categories = pgTable('categories', {
      id: text('id').primaryKey(),
      name: text('name').notNull().unique(),
      createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
    });

    export const products = pgTable('products', {
      id: text('id').primaryKey(),
      name: text('name').notNull(),
      categoryId: text('category_id').references(() => categories.id, { onDelete: 'set null' }),
      priceCmt: numeric('price_cmt').default('0'),
      hpp: numeric('hpp').default('0'),
      stock: numeric('stock').default('0'),
      isFavorite: boolean('is_favorite').default(false),
      updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
    });

    export const materials = pgTable('materials', {
      id: text('id').primaryKey(),
      name: text('name').notNull(),
      unit: text('unit').notNull(),
      price: numeric('price').default('0'),
      stock: numeric('stock').default('0'),
      updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
    });

    export const transactions = pgTable('transactions', {
      id: text('id').primaryKey(),
      itemId: text('item_id').notNull(),
      itemType: text('item_type').notNull(),
      type: text('type').notNull(),
      qty: numeric('qty').notNull(),
      date: timestamp('date', { withTimezone: true }).notNull(),
      notes: text('notes'),
      balanceAfter: numeric('balance_after').notNull(),
    });

    // ─── Better Auth Tables ───────────────────────────────────────────────────
    // Column names HARUS exact match dengan yang Better Auth buat di DB.
    // Better Auth v1 menggunakan camelCase column names.

    export const user = pgTable('user', {
      id: text('id').primaryKey(),
      name: text('name').notNull(),
      email: text('email').notNull().unique(),
      emailVerified: boolean('emailVerified').notNull().default(false),
      image: text('image'),
      createdAt: timestamp('createdAt').notNull().defaultNow(),
      updatedAt: timestamp('updatedAt').notNull().defaultNow(),
    });

    export const session = pgTable('session', {
      id: text('id').primaryKey(),
      expiresAt: timestamp('expiresAt').notNull(),
      token: text('token').notNull().unique(),
      createdAt: timestamp('createdAt').notNull().defaultNow(),
      updatedAt: timestamp('updatedAt').notNull().defaultNow(),
      ipAddress: text('ipAddress'),
      userAgent: text('userAgent'),
      userId: text('userId').notNull().references(() => user.id, { onDelete: 'cascade' }),
    });

    export const account = pgTable('account', {
      id: text('id').primaryKey(),
      accountId: text('accountId').notNull(),
      providerId: text('providerId').notNull(),
      userId: text('userId').notNull().references(() => user.id, { onDelete: 'cascade' }),
      accessToken: text('accessToken'),
      refreshToken: text('refreshToken'),
      idToken: text('idToken'),
      accessTokenExpiresAt: timestamp('accessTokenExpiresAt'),
      refreshTokenExpiresAt: timestamp('refreshTokenExpiresAt'),
      scope: text('scope'),
      password: text('password'),
      createdAt: timestamp('createdAt').notNull().defaultNow(),
      updatedAt: timestamp('updatedAt').notNull().defaultNow(),
    });

    export const verification = pgTable('verification', {
      id: text('id').primaryKey(),
      identifier: text('identifier').notNull(),
      value: text('value').notNull(),
      expiresAt: timestamp('expiresAt').notNull(),
      createdAt: timestamp('createdAt').defaultNow(),
      updatedAt: timestamp('updatedAt').defaultNow(),
    });
    ```

    **Avoid:**
    - Jangan ubah nama kolom DB — harus exact match dengan yang ada di NeonDB production
    - Jangan pakai `serial` atau `uuid` type untuk id — semua id adalah `text` (sesuai schema yang ada)
    - Jangan tambah kolom baru — ini harus match existing DB, bukan schema baru

<task type="auto">
  <name>Task 3: Rewrite lib/db.ts — export Drizzle instance (single Pool)</name>
  <files>lib/db.ts</files>
  <action>
    Ganti seluruh isi `lib/db.ts`. Hapus export `sql` (neon HTTP adapter) — tidak digunakan lagi setelah semua API direwrite dengan Drizzle di plan berikutnya.

    **Ganti SELURUH isi `lib/db.ts` dengan:**

    ```typescript
    import { drizzle } from 'drizzle-orm/neon-serverless';
    import { Pool } from '@neondatabase/serverless';
    import * as schema from './schema.js';

    const pool = new Pool({
      connectionString: process.env.DATABASE_URL,
      max: 10,
      connectionTimeoutMillis: 5000,
      idleTimeoutMillis: 10000,
    });

    export const db = drizzle(pool, { schema });
    ```

    **Kenapa ini benar:**
    - `drizzle-orm/neon-serverless` adalah adapter resmi Drizzle untuk `@neondatabase/serverless` Pool
    - `schema` disertakan agar Drizzle bisa melakukan relational queries (`db.query.products.findMany()`)
    - Satu Pool, satu `db` instance — dipakai oleh auth (Plan 04-02) dan semua API handlers (Plan 04-03, 04-04)
    - `max: 10` tetap dipertahankan sesuai NeonDB free tier limit

    **Avoid:**
    - Jangan export `sql` (neon HTTP adapter) lagi — ini yang diganti Drizzle
    - Jangan export raw `pool` — hanya `db` yang diekspos ke consumers
    - Jangan buat Pool di file lain (lib/auth.ts, api/auth-proxy.ts) — satu Pool via lib/db.ts saja
    - Jangan gunakan `drizzle-orm/neon-http` — itu untuk HTTP adapter, bukan Pool (kita butuh Pool untuk transactions)

    **Catatan:** `categories.ts` saat ini masih pakai `sql` dari lib/db.ts lama.
    Ini akan BREAK setelah plan ini. Ini expected — Plan 04-03 akan fix semua API endpoints.
    `lib/auth.ts` juga masih buat Pool sendiri — Plan 04-02 akan fix ini.
  </action>
  <verify>
