<objective>
## Goal
Rewrite `lib/auth.ts` agar gunakan `drizzleAdapter(db)` dengan `db` dari `lib/db.ts`, dan rewrite `api/auth-proxy.ts` agar import `auth` yang sama (single shared instance) — eliminasi duplicate Pool dan duplikasi `betterAuth()`.

## Purpose
Saat ini ada 3 Pool instances dan 2 `betterAuth()` calls di codebase (lib/auth.ts, api/auth-proxy.ts, plus pool di lib/db.ts). Ini adalah sumber dari auth yang tidak konsisten dan connection exhaustion risk. Plan ini menyatukan semuanya: satu Pool (via lib/db.ts), satu `db` (Drizzle instance), satu `auth` instance — dipakai oleh auth-proxy dan auth-middleware.

Ini juga mengaktifkan **drizzleAdapter** agar Better Auth pakai Drizzle queries ke NeonDB (bukan raw pg Pool) — konsisten dengan seluruh stack setelah Phase 04.

## Output
- `lib/auth.ts` — REWRITE: `drizzleAdapter(db, { provider: 'pg' })`, import `db` dari lib/db.ts
- `api/auth-proxy.ts` — REWRITE: import `auth` dari lib/auth.ts, hapus Pool dan betterAuth lokal
</objective>

  <name>Task 2: Rewrite api/auth-proxy.ts — import auth dari lib/auth.ts</name>
  <files>api/auth-proxy.ts</files>
  <action>
    Ganti seluruh isi `api/auth-proxy.ts`. Hapus local `Pool`, local `betterAuth()` — gunakan shared `auth` dari `lib/auth.ts`.

    Pertahankan seluruh logika handler yang sudah benar:
    - `export const config = { api: { bodyParser: false } }` — WAJIB, Vercel harus tidak parse body (toNodeHandler yang baca raw stream)
    - URL rewrite logic untuk query param `p` — needed untuk routing ke Better Auth actions
    - Origin header fallback — fix untuk Vercel CSRF check
    - Error handling dengan `res.headersSent` check

    **Ganti SELURUH isi `api/auth-proxy.ts` dengan:**
    ```typescript
    import { toNodeHandler } from 'better-auth/node';
    import { auth } from '../lib/auth.js';

    export const config = { api: { bodyParser: false } };

    const nodeHandler = toNodeHandler(auth);

    export default async function handler(req: any, res: any) {
      const p = req.query.p;
      if (p) {
        const pathStr = Array.isArray(p) ? p.join('/') : String(p);
        const decoded = decodeURIComponent(pathStr);
        const extra = Object.entries(req.query as Record<string, string>)
          .filter(([k]) => k !== 'p')
          .map(([k, v]) => `${k}=${encodeURIComponent(v)}`)
          .join('&');
        req.url = `/api/auth/${decoded}${extra ? '?' + extra : ''}`;
      }

      if (!req.headers.origin) {
        const proto = (req.headers['x-forwarded-proto'] as string) || 'https';
        const host = (req.headers['x-forwarded-host'] as string) || (req.headers.host as string);
        req.headers.origin = `${proto}://${host}`;
      }

      try {
        await nodeHandler(req, res);
      } catch (err: any) {
        console.error('[auth-proxy] error:', err?.message || String(err));
        if (!res.headersSent) {
          res.status(500).json({ error: 'auth-proxy error', detail: err?.message });
        }
      }
    }
    ```

    **Kenapa ini benar:**
    - Import `auth` dari `lib/auth.js` — satu instance, konsisten dengan lib/auth-middleware.ts
    - `toNodeHandler(auth)` dibuat satu kali di module level (tidak per request)
    - Semua logic yang sudah benar dipertahankan (URL rewrite, Origin fallback, error handling)

    **Avoid:**
    - Jangan import `betterAuth` atau `Pool` di file ini — tidak dibutuhkan lagi
    - Jangan hapus `export const config = { api: { bodyParser: false } }` — tanpa ini Vercel akan parse body dan toNodeHandler akan gagal (stream sudah empty)
    - Jangan pindah `toNodeHandler(auth)` ke dalam function handler — buat sekali di module level
    - Jangan ubah URL rewrite logic atau Origin fallback — ini sudah tested dan benar
  </action>
